Опис предметного середовіща

Розроблене програмне забезпечення призначене для користувачів, що пов’язані зі сферою логістики, а саме є представниками транспортних компаній, водіями, і т.д..

Завдяки розробленому програмному забезпеченню користувачі матимуть змогу переглядати типи послуг, отримувати основну інформацію про компанію та наявні типи машин, що використовуються для транспортування вантажів, створювати обліковий запис, а також надсилати запити компанії, що надає послуги.

Тема

Веб додаток для користувачів зі сфери логістики і вантажоперевезень.

Мета

За допомогою цього веб-додатку партнери матимуть змогу надсилати запити на траспортування вантажів по Америці, а також Мексиці і Канаді. Вони матимуть можливість переглянути інформацію стосовно своїх замовлень, оновлювати статус виконання перевезення, робити запити на оновлення локації і т.д..

Також, користувачі матимуть змогу переглядати інформацію стосовно компанії та відправляти форму для майбутньої співпраці з нею.

Тег <table> є контейнером для елементів, що >визначають вміст таблиці. Будь-яка таблиця складається з рядків і осередків, які задаються за допомогою тегів <tr> і <td>. Усередині <table> допустимо використовувати наступні елементи:<caption>, <col>,<caption>, <colgroup>, <tbody>, <td>, <tfoot>, <th>, <thead>, <tr>.

Тег <tbody> призначений для зберігання одного чи кількох рядків таблиці. Це дозволяє створювати структурні блоки, до яких можна застосовувати єдине оформлення через стилі, а також керувати їх видом через скрипти.

Тег <tr> служить контейнером для створення рядків таблиці.

Тег <td> Призначений для створення однієї комірки таблиці. Тег <td> повинен розміщуватися всередині контейнера <tr>, який в свою чергу розташовується всередині тега <table>.

Тег <br> встановлює новий рядок в тому місці, де цей тег знаходиться.

Тег <form> встановлює форму на веб-сторінці. Форма призначена для обміну даними між користувачем і сервером. Область застосування форм не обмежена відправкою даних на сервер, за допомогою клієнтських скриптів можна отримати доступ до будь-якого елементу форми, змінювати його і застосовувати на власний розсуд.

Тег <span> призначений для визначення малих елементів документа.

Тег <label> встановлює зв'язок між певною міткою, в якості якої зазвичай виступає текст, і елементом форми. Такая зв'язок необхідна, щоб змінювати значення елементів форми при натисканні курсором миші на текст.

Поле <textarea> є елементом форми для створення області, в яку можна вводити кілька рядків тексту. У текстовому полі допустимо робити переноси рядків, вони зберігаються при відправці даних на сервер.

Індефікатори

<section id="services"></section> <textarea name="message" cols="40" rows="10" id="message" > </textarea> <a class="lab" id="tableCode" href="#tableCodeFind">HTML-код таблиць</a> <a class="lab" id="formCode" href="#formCodeFind">HTML-код форми</a> <a class="lab" id="imgCode" href="#imgCodeFind">HTML-код зображення</a> <a class="lab" id="listCode" href="#listCodeFind">HTML-код списків</a>

Идентификатор (называемый также «ID селектор») определяет уникальное имя элемента,которое используется для изменения его стиля и обращения к нему через скрипты.

При описании идентификатора вначале указывается символ решётки (#), затем идет имя идентификатора.
Оно должно начинаться с латинского символа и может содержать в себе символ дефиса (-) и подчеркивания (_).

Использованиерусских букв в именах идентификатора недопустимо.

В отличие от классовидентификаторы должны быть уникальны, иными словами, встречаться в коде документа только один раз.

Відступи

th, td { padding: 15px; } #services { position: relative; height: 700px; width: 100%; padding-top: 25px; margin-bottom: auto; } .main-s { background-image:url('../img/Unknown.jpg'); min-height: 800px; margin-top: auto; padding-top: 50px; padding-left: 200px; padding-right: 200px; text-align: center; display: block; background-repeat: no-repeat; background-size: cover; } .input-row { margin: 0 0 40px; padding: 0 15px; position: relative; } .input-row { margin: 0 0 40px; padding: 0 15px; position: relative; } .x-row-inline >* { width: 40%; margin-left: auto; margin-right: auto; position: relative; top: 40px; } .input-row { margin: 0 0 40px; padding: 0 15px; position: relative; }

Свойство padding устанавливает внутренние отступы/поля со всех сторон элемента. Область отступов это пространство между содержанием элемента и его границей. Отрицательные значения не допускаются.

Свойство padding краткая форма записи свойств, чтобы не писать отдельное правило для каждой стороны (padding-top, padding-right, padding-bottom, padding-left).

Свойство margin CSS определяет внешний отступ на всех четырёх сторонах элемента.

Это сокращение, которое устанавливает все отдельные поля одновременно: (margin-top, margin-right, margin-bottom и margin-left)

Плаваючі елементи

<div style="float: left;" > <a class="text-link2" href="index.html">Go back</a> </div>

Плавающие элементы это такие элементы, которые обтекаются по
контуру другими объектами веб-страницы, например, текстом.

Следует понимать, что никто и нигде не плавает, поэтому правильнее говорить «обтекаемые элементы»

Свойство float CSS указывает, что элемент должен быть взят
из нормального потока и помещён вдоль левой или правой стороны его контейнера, где текст и встроенные элементы будут обтекать его.

Вісновки

У цій лабораторній роботі, ми навчилися працювати зі списками, вивчили відмінність між тегами маркірованих або нумерованих списків. Розібралися з індіфікатор і як їх використовувати. Навчилися працювати з плаваючими селекторами і зрозуміли, де зручно їх використовувати. Вивчили властивості css для відступів. Зрозуміли різницю між padding (внутрішні відступи) і margin (зовнішні відступи). Розібралися з видами селекторів і як їх використовувати.
